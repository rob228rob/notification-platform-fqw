\subsection{Укрупнённая схема потоков}

В текущем контуре используются три устойчивых класса потоков:
\begin{enumerate}
    \item события жизненного цикла уведомления;
    \item команды канальной доставки (немедленные и отложенные);
    \item статусные события доставки для контура истории.
\end{enumerate}

Формат интеграционного сообщения унифицирован, что упрощает трассировку и
поддерживает расширение набора событий без перестройки транспортного слоя.

\subsection{Инженерные свойства Kafka-слоя}

Использование Kafka в системе обеспечивает:
\begin{enumerate}
    \item разделение времени приёма команды и времени фактической обработки доставки;
    \item независимое масштабирование producer- и consumer-контуров;
    \item контролируемую повторную обработку при частичных отказах;
    \item наблюдаемость по producer- и listener-метрикам.
\end{enumerate}

Прикладная модель доставки остаётся at-least-once, поэтому защита от дублей
обеспечивается в прикладном слое через idempotency и ограничения данных
\cite{kafka_docs,kafka_design_semantics}.
